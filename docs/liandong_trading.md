## 1. 产品愿景

### 1.1 我们要解决的问题

**散户交易者的真正死因不是"看不准"，而是"管不住自己"。**

| 死因 | 占比 | 描述 |
|------|------|------|
| 重仓梭哈 | 35% | 一次性亏光本金 |
| 扛单不止损 | 30% | 小亏变大亏 |
| 频繁交易 | 20% | 手续费吃掉利润 |
| 追涨杀跌 | 15% | 情绪化操作 |

**核心洞察**：
> 散户不是输给市场，是输给自己。

### 1.2 我们的解决方案

**不是帮用户做更好的判断，而是——当用户想做蠢事时，系统说"不"。**

```
传统工具：给你更多信息 → 让你自己判断 → 你还是犯错
Liandong：限制你的错误 → 你想犯错也做不到 → 活下来
```

### 1.3 产品定位

| 维度 | 定位 |
|------|------|
| **品类** | 交易生存系统（不是交易工具） |
| **核心功能** | 限制错误行为（不是提供信息） |
| **用户感知** | "它在保护我"（不是"它在帮我赚钱"） |
| **终极价值** | 让用户活得更久（不是赚得更多） |

### 5.0 🧬 策略生命体（Liandong GPT 进化版）

> **核心理念**：策略不是静态代码，而是会自我进化的生命体

#### 5.0.1 什么是策略生命体

```
传统 GPT：用户输入需求 → 生成代码 → 完毕（静态）
策略生命体：用户输入需求 → 生成策略胚胎 → 策略自我进化 → 适者生存（动态）
```

#### 5.0.2 核心概念

| 概念 | 说明 |
|------|------|
| **策略 DNA** | 每个策略有基因编码（参数组合） |
| **自动变异** | 系统自动微调参数，适应市场变化 |
| **策略杂交** | 两个策略可以"交配"生成子策略 |
| **适者生存** | 表现差的策略自动"死亡"，好的策略繁殖 |

#### 5.0.3 进化路线

```
V1 参数优化      V2 遗传算法       V3 策略生态       V4 集体智慧

  参数组合        自然选择          UGC 生态        全网进化
```

#### 5.4.4 系统接管模式（Gatekeeper）

**核心理念**：不是给建议，是**直接限制行为**

**状态机**：
```
NORMAL（正常）
  ↓ 触发条件
RESTRICTED（受限）
  ↓ 继续违规
LOCKED（锁定）
  ↓ 冷静期 + 复盘
RECOVERING（恢复中）
  ↓ 24h 无违规
NORMAL（正常）
```

**各状态限制**：

| 操作 | NORMAL | RESTRICTED | LOCKED | RECOVERING |
|------|--------|------------|--------|------------|
| 开立现货 | ✅ | ✅ | ❌ | ⚠️ ≤10% |
| 开立合约 | ✅ | ❌ | ❌ | ⚠️ ≤2x |
| 加杠杆 | ✅ | ❌ | ❌ | ❌ |
| 加仓 | ✅ | ⚠️ ≤30% | ❌ | ⚠️ ≤10% |
| 减仓/平仓 | ✅ | ✅ | ✅ | ✅ |
| 设置止损 | ✅ | ✅ | ✅ | ✅ |

#### 5.4.5 高优先级补丁（规划中）

## 7. 生存率算法

### 7.1 完整算法

```python
def calculate_survival_rate(user_id, market_context):
    """
    计算用户的交易生存率
    
    生存率 = 风控配置分(40) + 系统完整度(30) + 行为表现(30)
    """
    
    # === 1. 风控配置分 (40分) ===
    config_score = 0
    
    # 是否绑定交易所 (+15)
    if has_exchange_binding(user_id):
        config_score += 15
    
    # 是否有交易系统 (+15)
    if has_active_protocol(user_id):
        config_score += 15
        
        # 是否有止损规则 (+10)
        if has_stop_loss_rules(user_id):
            config_score += 10
    
    # === 2. 系统完整度 (30分) ===
    system_score = 0
    
    # Guardian 是否启用 (+15)
    if is_guardian_enabled(user_id):
        system_score += 15
    
    # 是否有策略 (+15)
    if has_strategies(user_id):
        system_score += 15
    
    # === 3. 行为表现 (30分) ===
    behavior_score = 30  # 默认满分
    
    # 每次违规 -5分，最多扣20分
    violation_count = get_violation_count(user_id)
    behavior_score -= min(violation_count * 5, 20)
    
    # 连亏3次以上 -10分
    if consecutive_losses >= 3:
        behavior_score -= 10
    
    # === 总分 ===
    total = config_score + system_score + max(0, behavior_score)
    return max(0, min(100, total))
```
